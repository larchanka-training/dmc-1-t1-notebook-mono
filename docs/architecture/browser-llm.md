# Browser LLM — генерация текста на стороне клиента

## 1. Обзор

Markdown-ячейки поддерживают генерацию ответов с помощью языковой модели, запущенной **прямо в браузере** через [WebLLM](https://github.com/mlc-ai/web-llm). Вся инференция происходит на GPU пользователя через WebGPU — без сетевых запросов к бэкенду и без стоимости вызовов API.

Это принципиально отличается от генерации кода для code-ячеек, которая идёт через серверный пайплайн → AWS Bedrock (см. [`ai-generation.md`](./ai-generation.md)).

```
Markdown ячейка
     │  текст ячейки = промпт
     ▼
[WebLLMProvider]
     │  WebGPU / @mlc-ai/web-llm
     ▼
 Llama 3.2 1B (в браузере)
     │  сгенерированный текст
     ▼
 Следующая ячейка (тип raw)
```

## 2. Компоненты

### `useWebLLM.tsx` — контекст и хук

`WebLLMProvider` монтируется в `NotebookView` и немедленно начинает загрузку движка при открытии ноутбука. Хук `useWebLLM()` предоставляет:

| Поле | Тип | Описание |
|------|-----|----------|
| `status` | `"preparing" \| "ready" \| "error"` | Текущее состояние движка |
| `error` | `string \| null` | Сообщение об ошибке загрузки |
| `generate(prompt)` | `Promise<string>` | Отправить промпт в модель |

**Поведение `generate()` при вызове во время загрузки**: функция ожидает внутренний `readyPromise` — если пользователь нажал кнопку до завершения загрузки, запрос встаёт в очередь и выполняется сразу после готовности движка. Дополнительной логики очереди не требуется.

**Обработка ошибок**: при сбое загрузки `readyPromise` резолвится (не реджектится), `engineRef` остаётся `null`. Вызов `generate()` после этого бросает исходную ошибку загрузки, сохранённую в `loadErrorRef`.

### `BrowserLLMStatus.tsx` — индикатор в тулбаре

Отображается в правой части тулбара рядом с `KernelStatus` (JavaScript · idle/busy) и `ThemeToggle` (переключение тёмной/светлой темы):

| Статус | Индикатор |
|--------|-----------|
| `preparing` | Вращающийся спиннер |
| `ready` | Зелёная точка |
| `error` | Красная точка + слово «error» подчёркнуто пунктиром; при наведении — tooltip с текстом ошибки |

### Кнопка «Prompt AI» в `CellActions`

Появляется на markdown-ячейках (символ `✦`, tooltip «Prompt AI»). Кнопка отключена, если ячейка пуста или генерация уже идёт.

**Алгоритм при нажатии:**

1. Берёт `cell.source` (текст markdown-ячейки) как промпт
2. Вызывает `generate(prompt)` — при необходимости ждёт завершения загрузки
3. Смотрит на следующую ячейку в списке:
   - Если она типа `raw` — обновляет её содержимое
   - Иначе — создаёт новую `raw`-ячейку сразу после текущей и записывает результат туда

## 3. Модель

| Параметр | Значение |
|----------|----------|
| Модель | `Llama-3.2-1B-Instruct-q4f32_1-MLC` |
| Размер | ~800 MB |
| Библиотека | `@mlc-ai/web-llm` |
| Ускорение | WebGPU (Chrome 113+, Edge 113+) |

**Кэширование весов**: WebLLM использует браузерный Cache Storage API для хранения весов модели. Первая загрузка — ~800 MB; повторные визиты используют кэш. Очистить: DevTools → Application → Cache Storage.

---

## 4. Требования к окружению

### Браузер

WebLLM требует **WebGPU** — доступен в Chrome / Edge 113+ по умолчанию. Firefox и Safari не поддерживаются.

### Протокол: только HTTPS или localhost

Cache Storage API (используется WebLLM для кэширования весов) доступен только в **secure context**. На HTTP-соединении движок не загрузится.

Код проверяет `window.isSecureContext` перед инициализацией и немедленно устанавливает `status = "error"` с понятным сообщением, если условие не выполнено.

**Dev и prod окружения** обслуживаются через CloudFront, что обеспечивает HTTPS. Прямые HTTP URL ALB (`.elb.amazonaws.com`) для WebLLM не подходят. Подробнее: [`docs/dev-ops/aws-infrastructure.md`](../dev-ops/aws-infrastructure.md#cloudfront--https).
